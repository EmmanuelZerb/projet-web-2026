2. Detail des entites

2.1 UTILISATEURS

Table centrale du systeme. Represente tous les membres du reseau ECE In.

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
pseudoVARCHAR(50)NOT NULL, UNIQUENom d'utilisateur
emailVARCHAR(100)NOT NULL, UNIQUEEmail ECE (@edu.ece.fr ou @ece.fr)
mot_de_passeVARCHAR(255)NOT NULLHash bcrypt du mot de passe
nomVARCHAR(100)NOT NULLNom de famille
prenomVARCHAR(100)NOT NULLPrenom
roleENUM('auteur','administrateur')NOT NULL, DEFAULT 'auteur'Role dans le systeme
photoVARCHAR(255)DEFAULT avatarChemin vers l'avatar
image_fondVARCHAR(255)DEFAULT bgImage de couverture du profil
titreVARCHAR(200)NULLTitre professionnel
biographieTEXTNULLDescription personnelle
localisationVARCHAR(200)NULLVille, pays
site_webVARCHAR(255)NULLURL du portfolio
telephoneVARCHAR(30)NULLNumero de telephone
date_naissanceDATENULLDate de naissance
date_inscriptionDATETIMENOT NULL, DEFAULT NOW()Date de creation du compte
derniere_connexionDATETIMENULLDerniere activite
actifTINYINT(1)NOT NULL, DEFAULT 1Compte actif/desactive

2.2 FORMATIONS

Parcours academique d'un utilisateur.

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
utilisateur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEProprietaire
etablissementVARCHAR(200)NOT NULLNom de l'ecole/universite
diplomeVARCHAR(200)NOT NULLIntitule du diplome
domaineVARCHAR(200)NULLDomaine d'etude
date_debutDATENOT NULLDebut de la formation
date_finDATENULLFin de la formation
en_coursTINYINT(1)DEFAULT 0Formation en cours
descriptionTEXTNULLDetails supplementaires
lieuVARCHAR(200)NULLLocalisation

2.3 PROJETS

Projets realises par un utilisateur.

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
utilisateur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEProprietaire
titreVARCHAR(200)NOT NULLNom du projet
descriptionTEXTNULLDescription
date_debutDATENULLDebut
date_finDATENULLFin
en_coursTINYINT(1)DEFAULT 0Projet en cours
lienVARCHAR(255)NULLURL du projet (GitHub, etc.)
typeVARCHAR(100)NULLType (academique, personnel, entreprise...)

2.4 PUBLICATIONS

Contenu publie sur le fil d'actualite (statut, photo, video, CV, evenement, partage).

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
utilisateur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEAuteur
typeENUM('statut','photo','video','cv','evenement','partage')NOT NULL, DEFAULT 'statut'Type de publication
contenuTEXTNULLTexte de la publication
fichierVARCHAR(255)NULLChemin du fichier joint
fichier_mimeVARCHAR(100)NULLType MIME du fichier
lieuVARCHAR(200)NULLLocalisation
humeurVARCHAR(100)NULLSentiment
visibiliteENUM('public','amis','prive')NOT NULL, DEFAULT 'public'Niveau de visibilite
publication_originale_idINTNULL, FK → publications(id) ON DELETE SET NULLPublication partagee (auto-reference)
date_publicationDATETIMENOT NULL, DEFAULT NOW()Date de creation
date_modificationDATETIMENULLDerniere modification

2.5 REACTIONS

Reactions des utilisateurs sur les publications. Un utilisateur ne peut reagir qu'une fois par publication.

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
publication_idINTNOT NULL, FK → publications(id) ON DELETE CASCADEPublication reagie
utilisateur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEAuteur reaction
typeENUM('jaime','bravo','soutien','interessant')NOT NULL, DEFAULT 'jaime'Type de reaction
date_reactionDATETIMENOT NULL, DEFAULT NOW()Date de la reaction
Contrainte UNIQUE sur (publication_id, utilisateur_id) : une seule reaction par utilisateur et par publication.

2.6 COMMENTAIRES

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
publication_idINTNOT NULL, FK → publications(id) ON DELETE CASCADEPublication commentee
utilisateur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEAuteur du commentaire
contenuTEXTNOT NULLTexte du commentaire
date_commentaireDATETIMENOT NULL, DEFAULT NOW()Date

2.7 CONNEXIONS

Systeme d'amitie bidirectionnelle avec demande/acceptation.

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
demandeur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEPersonne qui envoie la demande
destinataire_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEPersonne qui recoit la demande
statutENUM('en_attente','accepte','refuse')NOT NULL, DEFAULT 'en_attente'Etat de la demande
date_demandeDATETIMENOT NULL, DEFAULT NOW()Date d'envoi
date_reponseDATETIMENULLDate de reponse
Contrainte UNIQUE sur (demandeur_id, destinataire_id) : une seule demande possible entre deux utilisateurs.

2.8 CONVERSATIONS

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
nomVARCHAR(200)NULLNom du groupe (si groupe)
est_groupeTINYINT(1)DEFAULT 0Conversation de groupe
date_creationDATETIMENOT NULL, DEFAULT NOW()Date de creation

2.9 CONVERSATION_PARTICIPANTS

Table d'association entre conversations et utilisateurs (cle primaire composite).

ChampTypeContraintesDescription
conversation_idINTPK, FK → conversations(id) ON DELETE CASCADEConversation
utilisateur_idINTPK, FK → utilisateurs(id) ON DELETE CASCADEParticipant
date_ajoutDATETIMENOT NULL, DEFAULT NOW()Date d'ajout

2.10 MESSAGES

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
conversation_idINTNOT NULL, FK → conversations(id) ON DELETE CASCADEConversation
expediteur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEExpediteur
contenuTEXTNOT NULLTexte du message
luTINYINT(1)DEFAULT 0Message lu par le destinataire
date_envoiDATETIMENOT NULL, DEFAULT NOW()Date d'envoi

2.11 NOTIFICATIONS

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
utilisateur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEDestinataire de la notification
expediteur_idINTNULL, FK → utilisateurs(id) ON DELETE SET NULLDeclencheur
typeENUM('demande_ami','ami_accepte','reaction','commentaire','partage','evenement','emploi','systeme')NOT NULLType de notification
messageTEXTNOT NULLTexte affiche
lienVARCHAR(255)NULLURL cible
lueTINYINT(1)DEFAULT 0Lue ou non
date_notificationDATETIMENOT NULL, DEFAULT NOW()Date de creation

2.12 EMPLOIS

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
publie_parINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEAuteur (admin)
titreVARCHAR(200)NOT NULLIntitule du poste
entrepriseVARCHAR(200)NOT NULLNom de l'entreprise
descriptionTEXTNOT NULLDescription du poste
typeENUM('cdi','cdd','stage','alternance','freelance','benevole')NOT NULLType de contrat
lieuVARCHAR(200)NULLLocalisation
salaireVARCHAR(100)NULLRemuneration
date_debutDATENULLDate de debut prevue
date_limiteDATENULLDate limite de candidature
lien_candidatureVARCHAR(255)NULLURL pour candidater
contact_emailVARCHAR(100)NULLEmail de contact RH
date_publicationDATETIMENOT NULL, DEFAULT NOW()Date de publication
actifTINYINT(1)DEFAULT 1Offre active

2.13 EVENEMENTS

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
organisateur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEOrganisateur
titreVARCHAR(200)NOT NULLTitre de l'evenement
descriptionTEXTNULLDescription
lieuVARCHAR(200)NULLLieu
date_debutDATETIMENOT NULLDate et heure de debut
date_finDATETIMENULLDate et heure de fin
imageVARCHAR(255)NULLImage de l'evenement
lienVARCHAR(255)NULLLien externe
est_officielTINYINT(1)DEFAULT 0Evenement officiel ECE
date_publicationDATETIMENOT NULL, DEFAULT NOW()Date de publication

2.14 ALBUMS

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
utilisateur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEProprietaire
nomVARCHAR(200)NOT NULLNom de l'album
descriptionTEXTNULLDescription
couvertureVARCHAR(255)NULLImage de couverture
date_creationDATETIMENOT NULL, DEFAULT NOW()Date de creation

2.15 ALBUM_PHOTOS

Table d'association entre albums et publications (photos). Cle primaire composite.

ChampTypeContraintesDescription
album_idINTPK, FK → albums(id) ON DELETE CASCADEAlbum
publication_idINTPK, FK → publications(id) ON DELETE CASCADEPublication photo

2.16 APPELS

Enregistrement des appels audio/video entre utilisateurs.

ChampTypeContraintesDescription
idINT AUTO_INCREMENTPRIMARY KEYIdentifiant unique
appelant_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEInitiateur de l'appel
receveur_idINTNOT NULL, FK → utilisateurs(id) ON DELETE CASCADEDestinataire de l'appel
conversation_idINTNOT NULL, FK → conversations(id) ON DELETE CASCADEConversation associee
typeENUM('audio','video')NOT NULL, DEFAULT 'audio'Type d'appel
statutENUM('sonnerie','en_cours','termine','refuse','manque')NOT NULL, DEFAULT 'sonnerie'Etat de l'appel
date_debutDATETIMENOT NULL, DEFAULT NOW()Debut de l'appel
date_finDATETIMENULLFin de l'appel
